# 🚀 앱 실행 방법

## ⚠️ 중요: 공유폴더 연결을 위해서는 HTTP 서버가 필수입니다!

`file://` 프로토콜(로컬 파일)에서는 File System Access API가 작동하지 않습니다.

---

## 🎯 3가지 방법 (쉬운 순서)

### 📌 방법 1️⃣: BAT 파일 더블클릭 (가장 간단)

**파일:** `START.bat`

**실행:**
```
데스크톱 또는 폴더 → START.bat 더블클릭
```

**결과:**
```
명령어 창이 뜨고
📍 http://localhost:8000 이라고 표시됨

← 브라우저에서 이 주소로 접근
```

✅ 공유폴더 연결 작동함

---

### 📌 방법 2️⃣: PowerShell로 실행

**파일:** `START.ps1`

**Step 1) Windows 검색에서 "PowerShell" 검색**
```
Windows 검색 → "PowerShell" → 클릭
```

**Step 2) 다음 명령어 복사 & 붙여넣기**
```powershell
cd "C:\Users\LOTTE\Desktop\업무보고 자동화"
python -m http.server 8000
```

**Step 3) Enter 키 누르기**

**결과:**
```
Serving HTTP on 0.0.0.0 port 8000 ...
```

✅ 공유폴더 연결 작동함

---

### 📌 방법 3️⃣: 직접 Python 명령어 (고급)

```bash
# 1. 명령 프롬프트 (cmd) 또는 PowerShell 열기

# 2. 폴더 이동
cd "C:\Users\LOTTE\Desktop\업무보고 자동화"

# 3. 서버 실행
python -m http.server 8000
```

✅ 공유폴더 연결 작동함

---

## 🌐 브라우저에서 열기

위 3가지 방법 중 하나로 HTTP 서버를 실행한 후:

**브라우저 주소창에 입력:**
```
http://localhost:8000
```

또는 Chrome/Edge를 열어서:
```
Ctrl+L → 
http://localhost:8000 입력 → Enter
```

공유폴더 연결 성공!

## 🐛 문제 해결

### "Python이 설치되지 않음" 오류
```
❌ python은 설치되지 않았습니다

해결:
1. Python 설치
   → https://www.python.org/downloads/
   → Windows 버전 다운로드
   → 설치 시 "Add Python to PATH" ✅ 체크

2. 설치 후 다시 START.bat 실행
```

### "포트 8000이 이미 사용 중" 오류
```
❌ Address already in use

해결:
1. 다른 포트 사용
   python -m http.server 9000  ← 9000으로 변경
   브라우저: http://localhost:9000

2. 또는 기존 프로세스 종료
   Ctrl+C (명령창에서)
   5초 대기
   다시 START.bat 실행
```

### "브라우저에서 폴더 선택 안 됨"
```
❌ [📁 폴더선택] 클릭해도 대화창 안 뜸

확인:
1. http://localhost:8000 접근했는가? (file:// 아님)
2. Chrome 또는 Edge인가? (Firefox 아님)
3. 주소가 정확한가?
```

### "설정 초기화되면서 다시 설정 화면 나타남"
```
❌ 설정 완료 → 로그인 → 새로고침하면 처음부터 시작

정상 동작입니다!
이유: localStorage는 http://localhost와 file://을 구분함

첫 로그인 후는 로그인 화면만 나타남 (설정 유지됨)
```

## ❓ 왜 HTTP 서버가 필요한가?

```
file:// 프로토콜
└─ 로컬 PC 파일 접근
   └─ 보안 제한: 폴더 읽기/쓰기 불가
   └─ 공유폴더 API 작동 ❌

http://localhost 프로토콜
└─ 로컬 웹 서버
   └─ 보안 허용됨
   └─ 공유폴더 API 작동 ✅ 
   └─ 공유폴더 읽기/쓰기 가능
```

즉, **HTTP 서버를 통해야만** 브라우저가 공유폴더에 접근 가능합니다.
